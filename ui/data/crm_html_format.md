# CRM HTML Formatting Reference

Tested against brtcrm.io on 2026-04-28.
The CRM uses a modern HTML sanitizer (HTMLPurifier) that allows a limited whitelist of inline tags and CSS properties.

---

## Allowed Tags

- `<strong>` вҖ” bold
- `<b>` вҖ” bold (same as strong)
- `<u>` вҖ” underline
- `<em>` вҖ” italic

## Allowed CSS Properties (via `style=`)

- `font-size: Npx`
- `color: name or hex`
- `font-style: italic`
- `text-decoration: line-through`
- `background-color: colorname`

## Other

- Actual newlines `\n` вҖ” line breaks
- `&nbsp;` вҖ” indentation/spacing
- Unicode characters вҖ” bullets, icons, emoji all render

## What Does NOT Work

- `<h1>`вҖ“`<h3>` heading tags
- `<ul>`, `<ol>`, `<li>` list tags
- `<br>` line break tag
- `<s>`, `<del>`, `<strike>` strikethrough tags
- `<sup>`, `<sub>` super/subscript
- `<span>`, `<div>`, `<p>` container tags
- `<font>` tag (HTML4)
- `<table border="">` вҖ” renders but no borders or column alignment
- `<details>`, `<summary>` collapsible
- `<blockquote>`, `<code>`, `<mark>`
- Markdown syntax (`#`, `**`, `-`, `>`, `` ` ``)
- `font-family: monospace` вҖ” stripped (no fixed-width column alignment possible)
- Tab characters `\t` вҖ” no column alignment
- Inline styles on `<span>`, `<div>`, `<p>` вҖ” tags stripped entirely
